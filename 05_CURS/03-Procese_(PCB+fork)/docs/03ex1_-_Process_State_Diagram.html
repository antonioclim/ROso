Transition Explanations

NEW → READY

Admitted — The process has been loaded into memory and is ready for execution. The PCB has been created and initialised.

READY → RUNNING

Dispatch — The scheduler has selected this process for CPU execution. A context switch takes place.

RUNNING → READY

Preempt — Time quantum expired or a higher priority process has arrived. The process returns to the queue.

RUNNING → WAITING

I/O Wait — The process is waiting for an I/O operation to complete (disk, network, etc.). The CPU becomes available.

WAITING → READY

I/O Complete — The I/O operation has finished. The process is once again eligible for execution.

RUNNING → TERMINATED

Exit — The process has completed execution (normally or with an error). Resources are released and the PCB is deleted.
