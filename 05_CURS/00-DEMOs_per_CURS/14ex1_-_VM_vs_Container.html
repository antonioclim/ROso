Când să folosești ce?

VM Ideal
  • • OS diferite (Linux + Windows)
  • • Izolare completă necesară
  • • Legacy applications
  • • Full system testing
  • • Security-critical workloads
VM Security
  • • Multi-tenant hosting
  • • Untrusted code execution
  • • Kernel-level isolation
  • • Compliance requirements
  • • Hardware-level separation
Container Ideal
  • • Microservices architecture
  • • CI/CD pipelines
  • • Rapid scaling
  • • Development environments
  • • Cloud-native apps
Container DevOps
  • • Fast iteration
  • • Kubernetes orchestration
  • • Resource efficiency
  • • Portable applications
  • • 12-factor apps

Tehnologii VM

Type 1 Hypervisor (Bare Metal)

Rulează direct pe hardware. Ex: VMware ESXi, Hyper-V, Xen

Type 2 Hypervisor (Hosted)

Rulează pe un OS host. Ex: VirtualBox, VMware Workstation

Hardware Virtualization

Intel VT-x, AMD-V - suport CPU pentru virtualizare eficientă

Tehnologii Container

Namespaces

Izolează: PID, Network, Mount, User, IPC, UTS

cgroups (Control Groups)

Limitează: CPU, Memory, I/O, Network bandwidth

Union File Systems

Overlay layers pentru imagini eficiente (OverlayFS, AUFS)

Abordare Hibridă: Best of Both Worlds

Containers in VMs

Rulează containers într-o VM pentru izolare suplimentară. Comun în cloud (EC2 + Docker).

Kata Containers

Lightweight VMs care rulează containere. Securitatea VM + viteza containerelor.

Firecracker

MicroVMs de la AWS. Boot în ~125ms, footprint minimal. Folosit de Lambda și Fargate.
