fork() creează o COPIE COMPLETĂ a spațiului de adrese (Copy-on-Write pentru optimizare). Procesele nu partajează memorie direct - comunicarea necesită IPC.

Thread-uri

Proces Container (PID: 2001) - Thread-uri Interne
Code Segment PARTAJAT
Data Segment PARTAJAT
counter = 0 Vizibil de toate thread-urile!
Heap PARTAJAT
File Descriptors PARTAJAT

Zone Private per Thread:

T1

Stack T1

Regs, TID: 1

Thread-urile partajează code, data, heap și file descriptors. Fiecare thread are doar stack și registre proprii. Creare: ~μs vs ~ms pentru procese.

Demonstrație: Comportamentul Variabilei Partajate

Scenariul 1: Threads

1. Variabilă globală: counter = 0

2. Thread 1 face counter++

3. Thread 2 vede imediat valoarea nouă!

Modificare vizibilă instant în toate thread-urile!

Scenariul 2: Procese

1. Variabilă globală: counter = 0

2. fork() → copilul modifică counter

3. Părintele NU vede modificarea!

Copii separate ale variabilei - izolare completă!
