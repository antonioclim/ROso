Hard vs Symbolic Links

Demonstrație interactivă a diferențelor dintre tipurile de link-uri

HARD LINKS

/home/user/
file.txt inode 100
/home/backup/
backup.txt inode 100
INODE 100
links: 2
data
Cum funcționează:

Ambele nume pointează direct la același inode. Sunt intrări separate în directoare diferite, dar referă aceleași date.

SYMBOLIC LINKS

/home/user/
file.txt inode 100
/home/
shortcut inode 200
INODE 100
links: 1
INODE 200
links: 1
"/home/user/file.txt"
Cum funcționează:

Symlink-ul are propriul inode care stochează doar un path string. La acces, sistemul urmărește path-ul.

Comenzi Linux

Hard Link

$ ln file.txt backup.txt
Creează hard link backup.txt → file.txt
$ ls -li file.txt backup.txt
Verifică: același inode number
$ stat file.txt | grep Links
Verifică link count

Symbolic Link

$ ln -s /home/user/file.txt shortcut
Creează symlink shortcut → file.txt
$ ls -l shortcut
Arată: shortcut -> /home/user/file.txt
$ readlink shortcut
Afișează target-ul symlink-ului
