TOT codul rulează în kernel mode

Avantaje

  • Performanță ridicată

    Comunicare directă între componente, fără overhead de message passing

  • System calls eficiente

    O singură tranziție User→Kernel pentru orice operație

Dezavantaje

  • Vulnerabil la crash

    Un bug în orice driver poate prăbuși întregul sistem

  • Complexitate mare

    Kernel-ul devine imens și greu de întreținut (Linux: ~30M linii cod)

Exemple Reale

Linux FreeBSD OpenBSD Classic UNIX MS-DOS

Avantaje

  • Foarte stabil

    Serviciile izolate - un crash în FS server nu afectează kernel-ul

  • Securitate sporită

    Suprafață de atac minimă în kernel, componente izolate

  • Ușor de extins

    Servicii noi pot fi adăugate fără modificarea kernel-ului

Dezavantaje

  • Performanță mai slabă

    Multiple tranziții IPC pentru operații simple

  • Overhead message passing

    Copiere date între spații de adrese diferite

Exemple Reale

MINIX 3 QNX seL4 Mach L4

Avantaje

  • Echilibru performanță/stabilitate

    Combină viteza monolithic cu modularitatea microkernel

  • Modularitate

    Drivere și servicii pot fi încărcate/descărcate dinamic

Dezavantaje

  • Complexitate arhitecturală

    Combinația de paradigme crește complexitatea

  • Bug-uri în kernel încă periculoase

    Componentele din kernel pot cauza crash

Exemple Reale

Windows NT/10/11 macOS (XNU) BeOS/Haiku
