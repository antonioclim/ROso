Cod Vulnerabil (C)

#include <string.h>
void vulnerable_func() {
char buffer[8];
int secret = 0;
gets(buffer); // UNSAFE!
if (secret != 0) {
grant_access();
}
}
Problema:

gets() nu verifică lungimea input-ului. Poate scrie dincolo de buffer!

Protecții Moderne

Stack Canary ON

Valoare specială între buffer și return address. Dacă e modificată → crash controlat.

ASLR ON

Address Space Layout Randomization. Adresele se schimbă la fiecare rulare.

DEP/NX Bit ON

Data Execution Prevention. Stack-ul e marcat non-executabil.

Safe Functions USE

Folosește fgets(), strncpy() în loc de gets(), strcpy().

Tipuri de Atacuri

1. Variable Overwrite

Suprascrie variabile adiacente (ex: flags, counters)

2. Return Address Hijack

Modifică adresa de return pentru a executa alt cod

3. Code Injection

Injectează și execută shellcode (blocat de DEP)

4. ROP (Return Oriented)

Înlănțuie "gadgets" existente pentru a ocoli DEP

Practici de Cod Sigur

NU folosiți:
gets() - niciodată bounds checking
strcpy() - poate overflow
sprintf() - format string + overflow
scanf("%s") - fără limită
Folosiți:
fgets(buf, size, stdin)
strncpy(dst, src, n)
snprintf(buf, size, fmt, ...)
scanf("%8s", buf) - cu limită
